SYSTEM ARCHITECTURE (WHAT TO REUSE)
-----------------------------------
1) UI shell in pywebview
   - HTML/CSS/JS frontend.
   - Python exposes a js_api bridge object.
   - JS calls Python methods for heavy work.

2) Bridge-centric orchestration
   - api/bridge.py is the single integration point.
   - It coordinates parse -> compute -> explain -> visualize.
   - Keep this pattern in your next app.

3) Worker isolation for rendering
   - Heavy/offscreen rendering runs in subprocess workers.
   - Avoids UI thread crashes/freezes and SDL conflicts.
   - This was one of the highest-value stability upgrades.

4) Data-driven learning content
   - Curriculum and knowledge are loaded from JSON payloads.
   - Rendering path is content-driven, not hardcoded.
   - Makes scaling chapter/slide libraries much easier.

5) Testing tiers
   - Unit + integration tests for logic
   - Smoke perf tests
   - E2E tests
   - Release checklist automation


BIG ISSUES WE RAN INTO (AND FIXES)
----------------------------------
1) pywebview + SymPy inspection warnings/noise
   Symptoms:
   - "_sorted_args: Atoms have no args"
   - expr_free_symbols deprecation warnings
   Cause:
   - pywebview reflects public object attributes deeply.
   Fix:
   - Keep SymPy-heavy internals private in bridge (`_parser`, `_solver`, etc.).
   - Use `expr.free_symbols` in parser output.
   - Reduce reflective exposure of complex objects.

2) pygame instability / trace trap / embedding friction
   Symptoms:
   - Runtime crashes and inconsistent behavior when rendering in-process.
   Fix:
   - Move slide rendering and capacity rendering into worker subprocesses.
   - Use offscreen env flags for workers (SDL dummy driver).
   - Return JSON payload/data-url back to bridge.

3) Missing ANTLR runtime for LaTeX parse
   Symptoms:
   - Warning from sympy latex parser.
   Fix:
   - Pin/install `antlr4-python3-runtime==4.11.1`.
   - Added in run.py dependency bootstrap.

4) Content JSON truncation/format drift
   Symptoms:
   - Curriculum incomplete parsing from content_jsons.txt.
   Fix:
   - Added robust extraction + minor repair + truncation fallback logic in bridge.
   - Maintains app resilience when content pipeline is mid-generation.

5) Slide text overflow / readability limits
   Symptoms:
   - Text clipped, unreadable, inconsistent capacity.
   Fix:
   - Built capacity worker and report generation.
   - Quantified usable chars per page with/without image.
   - Enabled content shaping decisions based on measured limits.

6) QA script dependency path issues
   Symptoms:
   - Missing setup script errors in full QA run.
   Fix:
   - Added fallback dependency installer path in run_full_qa.py.

7) Lint/type noise drowning signal
   Symptoms:
   - Too many low-value warnings early.
   Fix:
   - Focused lint gate on high-signal rules first (correctness).
   - Fixed mypy config and targeted type annotations.
   - Kept quality strict where it matters most.

IMPORTANT IMPLEMENTATION TIPS
-----------------------------
1) Keep bridge methods small and deterministic
   - Parse inputs
   - Call domain modules
   - Return JSON-safe payloads

2) Never render heavy graphics in UI thread
   - Use subprocess worker pattern from this app.

3) Make graph payloads generic and explicit
   Recommended shape:
   - success
   - x_range, y_range
   - curves[] / bars[] / points[] / annotations[]
   - legend[]
   - notes[]

4) Treat content overflow as a measured problem, not a guess
   - Reuse capacity worker/report logic.

5) Preserve copy/paste friendliness for all user-facing text
   - This was a major UX quality win.

6) Keep data and rendering decoupled
   - JSON content should drive UI without code rewrites.


KNOWN RISKS TO WATCH IN NEW DOMAIN
----------------------------------
- Large datasets causing slow payloads -> use paging/aggregation.
- Mixed metric scales in one chart -> define axis rules.
- Query ambiguity -> enforce canonical parse model + helpful error messages.
- Worker timeouts for heavy visual generation -> tune timeout + fallback states.

SNIPPET 3: SOLVE PIPELINE CONTRACT (api/bridge.py)
--------------------------------------------------
This method shape is highly reusable in other domains:

```python
def solve(self, latex_str: str, calc_type: Optional[str] = None, params: str = "{}") -> str:
    try:
        params_dict = json.loads(params) if isinstance(params, str) else (params or {})
        original_input = (latex_str or "").strip()

        detected = self._detector.detect(latex_str, calc_type or None)
        inner_latex, merged = self._extractor.extract(latex_str, calc_type, params_dict)

        parsed = self._parser.parse(inner_latex)
        if not parsed["success"]:
            parsed = self._parser.parse(latex_str)
        if not parsed["success"]:
            return json.dumps({"success": False, "error": parsed.get("error", "Parse failed")})

        expr = parsed["sympy_expr"]
        result = self._solver.solve(expr, detected, merged)

        if result["success"]:
            extracted = (inner_latex or "").strip()
            if extracted and original_input and extracted != original_input:
                result.setdefault("steps", [])
                result["steps"] = [{
                    "description": "Extract core expression from the original notation",
                    "before": original_input,
                    "after": extracted,
                    "rule": "context_extraction",
                }] + result["steps"]
            result["animation_steps"] = [s.to_dict() for s in self._step_gen.generate(result, detected)]
            result["result"] = str(result["result"])
            result["detected_type"] = detected.name

        return json.dumps(result, default=str)
    except Exception as e:
        return json.dumps({"success": False, "error": str(e)})
```

How to adapt for football:
- `detect` -> detect query type (`PLAYER_COMPARE`, `TEAM_TREND`, etc.)
- `extract` -> parse filters (season/week/team/position)
- `parse` -> produce canonical query object
- `solver` -> compute analytics
- `step_gen` -> explanation steps ("Filtered season", "Computed EPA/play", etc.)

```python
def _paginate(text: str, font, max_width: int, max_lines: int) -> list[str]:
    paras = text.replace("\r\n", "\n").split("\n")
    all_lines = []
    for p in paras:
        if p.strip():
            all_lines.extend(_wrap_paragraph_lines(p, font, max_width))
        else:
            all_lines.append("")
    pages = []
    for i in range(0, len(all_lines), max_lines):
        pages.append("\n".join(all_lines[i:i + max_lines]).strip("\n"))
    return pages or [""]
```
